# 學術翻譯方法論

程式碼處理「版面」，這份文件處理「翻譯本身」。學術翻譯的共同目標：
**忠實、精確、術語零歧義、可對回原文**，而非文學再創作。

## 先判斷文本類型與語域（動筆前第一件事）

本 skill 同時服務兩類文本，**語域不同，策略不同**：

| 類型 | 例子 | 語域策略 |
|------|------|----------|
| **現代技術論文** | ML/醫學/工程期刊、會議論文 | 精確簡潔、術語照標準譯法、被動轉主動、引用/數值/公式原樣保留。本文以下各節主要針對這類。 |
| **人文/社科經典** | 哲學論說文、歷史文獻、思想史（如 Spencer《教育論》、Mill、Dewey）| **思辨流暢、保留論證節奏與修辭**。長句層層推演可拆成中文短句，但不可打散邏輯連接（「不僅…而且…」「與其說…不如說…」要譯出）。反諷、類比、排比要保留力道。術語是**概念詞**（如 intrinsic worth、complete living、discipline）而非技術名詞，鎖定後全書一致。 |

人文經典**沒有圖表/公式/參考文獻清單**（Phase 3/4 及「參考文獻不翻」規則多半用不到），
重點在**語感與概念一致性**。以下「被動語態、術語標準譯法、常見陷阱」各節，技術類全適用，
人文類擇要參考（術語一致性、反翻譯腔、被動處理仍通用；SOTA/ablation 等技術詞則無關）。

## 敏感史料（翻譯前先知道）

19 世紀以前的人文經典常含當時的殖民、種族、性別用語（savage、barbarian、wild tribes、
aboriginal 等）。**照原文忠實翻譯**，不美化也不刪節——這是史料的一部分。若代管 API 的輸出
過濾器誤擋，見 `runbook.md` 的對策（縮小輸出塊、直接寫檔、或換過濾較寬鬆的模型跑翻譯步驟）。

## 論文翻譯的第一原則：精確 > 通順 > 優雅

- 寧可句子略生硬，也不能改變技術含義。
- 不確定的專有名詞，寧可保留原文，也不要猜一個中文。
- 數字、單位、符號、引用標記一律照抄，錯一個數字整段作廢。

## 三種文體，三種譯法

| 文體 | 例子 | 譯法 |
|------|------|------|
| **技術陳述**（方法、結果） | "We trained the model for 100 epochs." | 精確、簡潔、主動語態。術語照詞庫。 |
| **論證/討論** | "This suggests that ... may be attributed to ..." | 保留推論的謹慎語氣（suggests→顯示/暗示，不要斷定）。 |
| **摘要/引言的宏觀敘述** | "Recent advances have revolutionized ..." | 可通順化，但不誇大；學術中文避免行銷腔。 |

## 被動語態的處理（論文最常見的翻譯腔來源）

英文學術寫作大量用被動（"was observed", "is defined as"）。中文不必跟著被動：
- "The temperature was measured" →「我們量測溫度」或「溫度經量測為…」，
  而非生硬的「溫度被量測」。
- 但**方法段落**保留客觀性時，可用「經…」「以…」的無主句，不要硬塞「我們」。

## 什麼要翻、什麼保留原文

| 內容 | 處理 |
|------|------|
| 正文、摘要、圖說、表格內文字 | 翻譯 |
| **領域術語首次出現** | `中文<span class="en">English</span>`，**無論純中或對照版都附**（查回原文是剛需）|
| 引用標記 `[12]`、`(Smith et al., 2020)` | **原樣保留，絕不改動** |
| 數值、單位、統計量 `p<0.05`、`95% CI` | 原樣保留 |
| 數學符號、變數名 `x_i`、`θ`、`∑` | 保留；獨立公式用截圖（見 SKILL Phase 4）|
| 模型/方法/資料集專名 `BERT`、`ImageNet` | 通常**保留原文不譯** |
| 縮寫 `CNN`、`RL` | 首見可「中文（全稱＋縮寫）」，之後用縮寫 |
| 作者名、機構、期刊名 | 正文提及可音譯附原文；**參考文獻整段不翻** |
| 參考文獻清單 | **完全不翻**，原樣保留 |
| **詩歌／引詩（epigraph、古謠）** | 用 `.poem` 結構；**中英並列**（`.line.zh` + `.line.orig`），保留詩行斷句。若原詩格律難譯，寧可保留原文並附白話中譯 |
| 致謝、利益聲明、資助資訊 | 可翻，也可原樣（依需求）|

## 術語標準譯法：先查，再鎖，必要才問

論文術語多為領域既有詞，**不要自己生造**。Phase 0 對每個核心術語：

1. **WebSearch 查主流中譯**：中文維基、教育部學術名詞、領域教科書、知名中文技術部落格。
2. 決策：
   - **唯一主流譯法** → 直接鎖定（如 gradient descent→梯度下降、overfitting→過擬合/過度配適）。
   - **兩岸用語差異** → 選台灣繁中慣例（如 optimization 台灣多用「最佳化」而非「优化」；
     information 台灣「資訊」非「信息」；video「影片」非「视频」；algorithm「演算法」非「算法」）。
   - **≥2 種台灣主流譯法並存，或論文自造新詞（coined term）** → 放「待確認」，一次問使用者。
3. 鎖定後全文一致，遇到就照用，不重新決定。

**兩岸常見差異對照（一律取台灣繁中）：**

| 概念 | 台灣（用這個） | 對岸（避免） |
|------|--------------|-------------|
| optimization | 最佳化 | 优化 |
| algorithm | 演算法 | 算法 |
| information | 資訊 | 信息 |
| video | 影片 | 视频 |
| default | 預設 | 默认 |
| object | 物件 | 对象 |
| memory | 記憶體 | 内存 |
| performance | 效能 | 性能 |
| robust | 穩健/強健 | 鲁棒 |

## 反翻譯腔檢查清單

翻完一段回讀，問：
- [ ] 這句話中文母語的研究者會這樣寫論文嗎？
- [ ] 被動式是否該轉主動或無主句？
- [ ] 「的的不休」？連續「的」拆掉。
- [ ] 英文長句的 which/that/关系子句，是否拆成中文短句？
- [ ] 引用標記、數字、單位、符號原樣保留？
- [ ] 術語是否用了詞庫鎖定的譯法（沒有同詞多譯）？
- [ ] 有沒有把 "significant" 一律翻「顯著」而忽略統計 vs 一般語意？

## 常見陷阱

- **significant**：統計顯著（p 值語境）vs 一般「重要/可觀」，看上下文分別譯。
- **novel / naive**：novel=新穎的（非「小說」）；naive=樸素的/未加處理的（非「天真」）。
- **state of the art (SOTA)**：最先進、當前最佳，別逐字「藝術狀態」。
- **ablation study**：消融實驗/消融研究（固定譯名）。
- **ground truth**：真實標註／基準真值（領域固定）。
- **baseline**：基準線／基準方法。
- 中文全形標點 vs 公式內半形：**公式、程式碼、變數維持半形**；正文用全形。

## 品質保證流程

1. **逐節自審**：翻完一節用上面清單通讀。
2. **術語掃描**：用 glossary-guide.md 的一致性腳本，確認沒有同詞多譯。
3. **對照原文抽查**：隨機抽幾段開「中英對照」檢視，確認沒漏譯、數值沒抄錯、引用標記對得上。
4. **符號完整性**：公式截圖是否都成功內嵌、編號對得上正文。
