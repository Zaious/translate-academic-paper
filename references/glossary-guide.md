# 詞庫建立與管理指南（論文版）

## 為什麼論文更需要詞庫

一篇論文同一術語出現數十次。若 "representation" 在方法段翻「表徵」、結果段翻「表示」、
討論段翻「呈現」，讀者會以為是三個概念。詞庫的功能是**每個術語全文只有一個中文對應**。

論文與 TRPG 的關鍵差異：**論文術語多是領域既有詞，要「查標準譯法」而非「創譯」。**

## 建立流程

### 步驟 1：收集候選術語

翻譯前，用 PyMuPDF 抽出標題、摘要、關鍵詞、重複出現的名詞片語。摘要與引言的術語密度最高。

```python
import fitz, re
from collections import Counter
doc = fitz.open("paper.pdf")
# 全大寫縮寫 + 首字大寫的名詞片語常是術語
text = "\n".join(p.get_text() for p in doc)
acronyms = Counter(re.findall(r'\b[A-Z]{2,}\b', text))
print(acronyms.most_common(30))
```

### 步驟 2：查標準譯法（論文版核心步驟）

對每個核心術語 **WebSearch** 查主流中譯，來源優先序：
1. 教育部學術名詞資訊網
2. 該領域中文教科書 / 中文維基對應條目
3. 知名中文技術社群慣例

決策規則：
- **唯一主流譯法** → 直接鎖定，不打擾使用者。
- **兩岸差異** → 取台灣繁中（見 methodology.md 對照表）。
- **≥2 種台灣譯法並存，或論文自造新詞** → 放「待確認」，翻該章前一次問使用者。

### 步驟 3：分三層

**第一層：領域核心術語**（翻正文前鎖定）——這篇論文反覆出現、決定理解的詞。
例：attention→注意力、regularization→正則化、gradient descent→梯度下降。

**第二層：本文自訂概念 / 模型名 / 縮寫**——作者提出或命名的東西。
- 模型/方法/系統專名：**多半保留原文**（BERT、Transformer、ResNet 不譯）。
- 作者自造的概念詞：查無標準譯法 → 進「待確認」問使用者，鎖定後附原文。

**第三層：人名 / 機構 / 資料集 / 工具名**——
- 資料集、工具、函式庫名：**保留原文**（ImageNet、PyTorch、GLUE）。
- 人名在正文提及：音譯附原文；參考文獻裡的作者名**不翻**。

### 步驟 4：「待確認」區

```markdown
## 待確認
| English | 候選中文 A | 候選中文 B | 問題 |
|---------|-----------|-----------|------|
| embedding | 嵌入 | 詞向量 | 泛指 embedding 用「嵌入」，特指 word embedding 才「詞向量」？ |
| latent | 潛在 | 隱 | latent variable→潛在變數 vs latent space→隱空間，要統一嗎？ |
```

翻完該節再批次決定，別邊翻邊猜。

### 步驟 5：每節翻完更新詞庫

新術語加入對應層；解決的「待確認」移到對應層；發現舊決定不對記在「修訂記錄」。

## 詞庫格式

```markdown
# 翻譯詞庫 — 《論文標題》

## 領域與主題摘要
（150-300 字：哪個領域、方法論流派、核心貢獻、語域基調。這是全文用詞的錨點。
 例：本文屬深度學習 NLP，提出一種預訓練語言模型。用詞取台灣繁中資訊領域慣例，
 學術中性語域，模型與資料集名保留原文。）

## 第一層：領域核心術語
| English | 中文 | 說明 | 來源 | 鎖定 |
|---------|------|------|------|------|
| attention | 注意力 | 核心機制 | 中文維基 | ✓ |
| fine-tuning | 微調 | | 教科書慣例 | ✓ |
| overfitting | 過擬合 | 台灣亦用「過度配適」 | | ✓ |

## 第二層：本文自訂概念 / 模型名 / 縮寫
| English | 中文 | 處理 | 首見 |
|---------|------|------|------|
| BERT | BERT | 保留原文 | 摘要 |
| Masked LM | 遮罩語言模型（Masked LM） | 意譯附原文 | §3 |

## 第三層：人名 / 資料集 / 工具
| English | 中文 | 備註 |
|---------|------|------|
| GLUE | GLUE | 保留原文（基準名）|
| ImageNet | ImageNet | 保留原文 |
```

## 術語一致性檢查

翻完幾節後掃 HTML，抓同詞多譯：

```python
import re
from pathlib import Path
CHECK_PAIRS = [("表徵", "表示"), ("最佳化", "优化"), ("潛在", "隱")]
for f in sorted(Path("build").glob("sec*.html")):
    t = f.read_text(encoding="utf-8")
    for a, b in CHECK_PAIRS:
        if a in t and b in t:
            print(f"⚠ {f.name}: 同時出現「{a}」和「{b}」")
```

## 常見錯誤

| 錯誤 | 為什麼不好 | 怎麼改 |
|------|-----------|--------|
| 自己生造術語譯名 | 領域已有標準譯法 | 先 WebSearch 查 |
| 沿用對岸譯名 | 台灣讀者不慣（优化/鲁棒/信息）| 取台灣繁中慣例 |
| 模型/資料集名硬翻中文 | BERT 翻「伯特」沒人看得懂 | 專名保留原文 |
| 邊翻邊決定術語 | 同詞多譯 | 先建詞庫再翻 |
| 翻參考文獻 | 破壞可檢索性、易出錯 | 整段原樣保留 |
